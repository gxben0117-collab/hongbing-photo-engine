# 核心咒語保護契約

## 目的

本文件定義哪些內容屬於核心咒語規範。除非專案 owner 明確同意，維護時不得修改這些區塊。

## 受保護範圍

- 身份鎖定文字
- 骨架、五官、臉型與辨識度約束
- 各頁主要 prompt data 常數
- 最終咒語輸出的段落順序
- 主題、構圖、姿態、鏡頭、光線、媒材的英文提示詞內容
- 預設值造成的核心語意變化

## 允許先行處理的低風險範圍

- 文件補齊與文件順序修正
- 靜態檢查腳本
- 不改變 DOM 結構語意的 CSS 整理
- 不改變輸出文字的 UI helper 重構
- 本地連結、重複 id、語法檢查

## 高風險變更規則

以下變更必須先回報並取得同意：

- 修改 prompt 文字
- 新增、刪除或重排最終輸出段落
- 調整會自動覆寫使用者選項的預設聯動
- 將 inline prompt data 搬移到外部資料檔
- 修改使用者已熟悉的操作順序

## 驗收原則

- 修改前後必須能說清楚「咒語輸出是否完全相同」
- 若輸出會改變，必須先取得 owner 同意
- 上架前至少執行靜態檢查、連結檢查、重複 id 檢查與基本 UI 流程檢查
