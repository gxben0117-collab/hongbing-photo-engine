# 動漫合鏡系統介面與因果稽查：改前／改後

## 稽查範圍

檢查主模式、角色原型、替身／式神細節、關係敘事與隨機世界觀之間是否會產生矛盾，並以高相似度鎖臉與第二存在身份隔離為不可違反條件。

## 發現與修正

| 稽查項目 | 改前 | 改後 |
| --- | --- | --- |
| 變身 | 文案與咒語可理解成主角旁邊站另一名完成型英雄 | 固定為同一主角的前後變身層，禁止獨立隊友／第二張人臉 |
| 替身 | 可選守護替身，但未必顯示對應細節 | 守護替身模式直接帶入形態與能力，並要求由主角能量顯現 |
| 機甲 | 容易成為機甲與人並站的普通合照 | 固定主角核心／指令連線與機甲胸口核心同步，限定為機械存在 |
| 式神 | 只有選靈媒原型時才將式神資料寫進咒語 | 選式神模式即寫入式神形態與契約，並在 UI 顯示相應欄位 |
| 隨機 | 各欄位獨立抽選，可能產生互相矛盾的設定 | 從已調校的一鍵世界觀抽取整組設定 |

## 生成順序

高相似度模式的固定順序維持：主角身份核心 → 高相似度鎖臉 → 第二存在身份隔離 → 關係因果 → 主模式執行規則 → 角色、姿勢、背景與特效。
