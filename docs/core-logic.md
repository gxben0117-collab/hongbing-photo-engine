# 核心咒語邏輯

## 共用原則

- 身份鎖定優先於風格變化：任何頁面、任何選項都不能覆寫身份鎖定與臉部幾何。
- 構圖、姿態、鏡頭、背景、服裝、光線分層組合，段落順序固定（詳見「輸出段落順序」）。
- 預設值只負責補齊，不覆寫使用者的核心意圖；「自動 / auto」永遠是安全預設，
  等於「不輸出這段限制，交給模型自己決定」，不是「隨機選一個」。
- 最終輸出保持英文（少數中文段落標題如「【主題】」是給人看的分節標記，不影響模型判讀）。
- 完整的身份保護語意細節見 `docs/shared-protection-core.md`；能不能改這些文字見
  `docs/core-prompt-contract.md`。

## 共用核心（`assets/core-prompt.js`）

`window.HB_CORE_PROMPT` 集中管理五頁共用的區塊：

| 區塊 | 用途 |
| --- | --- |
| `identityLock` | 身份鎖定（最高優先級） |
| `faceGeometryLock` | 臉部幾何鎖定 |
| `realisticAnatomy` | 真人骨架系統 |
| `cameraReconstruction` | 鏡頭重建（忽略原始自拍透視） |
| `lightingUnification` | 光線一致性系統 |
| `skinTexture` | 真實膚質系統 |
| `illustrationReconstruction` | 插畫媒材版的骨架重建（見下方「插畫媒材條件化」） |
| `negativePrompt` | 通用負面約束 |
| `outputQuality` | 輸出規格（4K、銳利對焦等） |
| `poseNaturality` | 姿勢自然性系統 |
| `cleanFrame` | 畫面淨化（單人、無雜訊） |

`page.travel` / `page.magazine` / `page.fantasy` / `page.doll` / `page.storeAd`
是把上面的區塊組合成各頁需要的子集（例如 doll 不需要 `poseNaturality`，
store-ad 只在人物照模式才掛 `identityLock`）。

## 插畫媒材條件化

travel、magazine、fantasy 三頁都有「寫實 vs 插畫」的媒材/材質選項（例如
travel 的「水彩旅行插畫」、magazine 的「日系動畫畫風」、fantasy 的紙雕/摺紙/
水墨材質）。選到插畫類時，組裝邏輯會把預設的寫實骨架（含「Visible fine pores」
等寫實膚質字句）換成 `illustrationSkeleton`（不要求照片級毛孔細節，但仍要求
臉部幾何可辨識）——避免同一份咒語裡同時出現「畫成水彩插畫」和「拍出毛孔細節」
這種互相矛盾的指令。

判斷邏輯集中在每頁自己的 `xxx_ILLUSTRATION_MEDIA_KEYS` / `FANTASY_ILLUSTRATION_MATERIAL_KEYS`
這類 Set，新增插畫類選項時記得把 key 加進對應的 Set。

## 輸出段落順序（以 travel 為例，其他頁同一套精神）

1. 身份鎖定 → 2. 骨架（寫實或插畫版）→ 3. 光線一致 → 4. 姿勢自然性 →
5. 攝影師模組 → 6. 光線細節 → 7. 鏡頭 → 8. 姿勢模組（若非 auto）→
9. 動態感 → 10. 構圖（若鏡頭未覆蓋構圖，見下方例外）→ 11. 風格 → 12. 媒材 →
13. 主題 → 14. 服裝邏輯 → 15. 裝扮細節（若有選）→ 16. 畫面淨化 →
17. 輸出規格 → 18. 輸出比例

這個順序受 `core-prompt-contract.md` 保護，不可任意重排。

**已知例外**：travel 的鏡頭選項裡「雜誌封面感／廣角環境感／遠距主視覺」
本質是構圖用語（跟 01 構圖同一個軸），選到這三者時會跳過構圖段落，
避免同時輸出兩種互相矛盾的景別指令。

## 生成 → 顯示 → 複製（travel / magazine / fantasy 共用）

三頁的操作模式已在 2026-07-19～21 統一（見 `docs/development-log.md`）：

- **手動生成**：改選項不會即時重算，要按「生成完整咒語」才會組裝並顯示。
- **Stale 保護**：生成過一次後，只要再改任何選項，輸出區就標記為過期
  （文字轉淡、金框提示、複製鈕失效），直到重新按生成才清除。目的是避免使用者
  複製到一份跟畫面目前選項對不上的舊咒語。
- **套用即顯示**：不管是具名的「一鍵套用｜XXX」按鈕，還是「隨機套用」按鈕，
  套用後都會立即生成、顯示、捲動到結果，不需要再手動按一次生成鈕。

`doll.html` 目前還沒套用這套規則（手動生成、無 stale 保護、一鍵主題不會自動顯示）。
`store-ad.html` 是完全不同的設計（表單邊填邊即時看到海報企劃，沒有「生成」這個
獨立動作），不適用這套規則。

## 隨機套用：元素級獨立隨機

三頁的「隨機套用｜⋯」按鈕（2026-07-22 起）都是**每個欄位各自從完整選項池獨立抽選，
再動態組合成一份結果**，不是從一批預寫好的完整組合裡三選一：

- travel：構圖、風格、比例、鏡頭、光線、動態、媒材、姿勢、服裝、髮型、配件、
  主題文字，每一項各自獨立隨機。
- magazine：風格、背景、姿態、框景、鏡頭、動態、媒材、比例、妝容、珠寶、
  表情、光線，每一項各自獨立隨機。
- fantasy：服裝、材質、背景、光線、構圖、框景、姿勢、風格、鏡頭、比例、強度，
  每一項各自獨立隨機。

實作靠 `getAllRadioValues(name)` / `getAllSelectValues(id)` 直接讀取當下 DOM 上
存在的選項清單，所以之後新增任何選項，隨機池會自動吃進去，不需要回頭改隨機函式。

三頁既有的具名「一鍵套用｜XXX」模板/預設（travel 11 個、magazine 20 個、
fantasy 40 個）是刻意保留的「設計師精選組合」，跟隨機按鈕是兩套不同的東西，
新增/修改具名模板不影響隨機邏輯，反之亦然。

## 版權角色名規則

讀取風格參考圖時經常會遇到遊戲/動漫角色的 cosplay 圖（例如不知火舞、月野うさぎ、
胡蝶しのぶ）。這類圖片**只能抽取視覺技法**（材質、構圖、姿勢、光線），角色名、
作品名、可辨識的角色專屬設計一律不得進入 prompt 文字或 UI 顯示文字。
`scripts/audit-100x.mjs` 有一份已知角色名的靜態掃描名單，新增選項後務必重跑一次。

## 咒語內容檢查工具

- `scripts/check-static.mjs`：結構檢查（重複 id、本地連結、inline script 語法）。
- `scripts/build-prompt-preview.mjs`：用 `node:vm` 重建改前/改後的完整咒語，
  比對固定選項組合的輸出是否有變化（0 diff = 純行為/UI 調整，沒有動到 prompt 文字）。
- `scripts/audit-100x.mjs`：重建五頁的實際組裝邏輯，每頁隨機抽 100 組選項模擬，
  檢查 `undefined`/`NaN`/`[object Object]` 洩漏、身份鎖定是否存在、相鄰重複行、
  禁用角色名靜態掃描。新增選項或改組裝邏輯後都應該重跑。

三個腳本合起來覆蓋「結構對不對」「舊輸出有沒有被動到」「新輸出有沒有內容問題」
三個不同的驗證面向，任何一波咒語相關的改動都應該三個都跑過。
